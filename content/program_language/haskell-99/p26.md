# P26 - 生成所有 K 元组合

> Generate the combinations of K distinct objects
>
> 官方模块：`Problems.P26`　核心函数：`combinations`

## 题目描述

从 $n$ 个列表元素中选择 $k$ 个，生成所有组合。组合保留输入顺序，不把同一批元素的不同排列重复计数。

## 函数签名

```haskell
combinations :: Int -> [a] -> [[a]]
```

## 思路

面对首元素 `x`，每个组合只有两类：

1. 不选择 `x`，继续从 `xs` 中选择 $k$ 个；
2. 选择 `x`，再从 `xs` 中选择 $k-1$ 个。

这正是二项式恒等式 $\binom{n}{k}=\binom{n-1}{k}+\binom{n-1}{k-1}$ 的程序版本。

## 实现

```haskell
combinations :: Int -> [a] -> [[a]]
combinations 0 _      = [[]]
combinations k _ | k < 0 = []
combinations _ []     = []
combinations k (x:xs) =
  combinations k xs ++ map (x :) (combinations (k - 1) xs)
```

`[[]]` 和 `[]` 不同：选择 0 个元素有一种方案，即空组合；无法选择时才是零种方案。

## 测试

```haskell
>>> combinations 2 "abcd"
["cd","bd","bc","ad","ac","ab"]
>>> length (combinations 3 [1..12])
220
>>> combinations 0 [1,2,3]
[[]]
>>> combinations 4 [1,2,3]
[]
```

## 复杂度

结果本身有 $\binom{n}{k}$ 个，每个结果长度为 $k$，因此仅构造输出就需要 $O(k\binom{n}{k})$ 的时间和空间。
