# P26 - 生成所有 K 元组合

> Generate the combinations of K distinct objects chosen from the N elements of a list
>
> 官方模块：`Problems.P26`　核心函数：`combinations`

---

## 题目描述

从 $n$ 个列表元素中选择 $k$ 个，生成所有组合。组合保留输入顺序，不把同一批元素的不同排列重复计数。

## 函数签名

```haskell
combinations :: Int -> [a] -> [[a]]
```

## 实现

### 方法一：递归回溯

```haskell
combinations :: Int -> [a] -> [[a]]
combinations 0 _      = [[]]
combinations k _ | k < 0 = []
combinations _ []     = []
combinations k (x:xs) =
  combinations k xs ++ map (x :) (combinations (k - 1) xs)
```

面对首元素 `x`，每个组合只有两类：
1. **不选** `x`：从 `xs` 中选 $k$ 个
2. **选** `x`：从 `xs` 中选 $k-1$ 个，然后 `x` 放在最前面

这正是二项式恒等式 $\binom{n}{k}=\binom{n-1}{k}+\binom{n-1}{k-1}$ 的程序版本。

### 方法二：列表推导

```haskell
combinations :: Int -> [a] -> [[a]]
combinations 0 _      = [[]]
combinations k xs
  | k < 0     = []
  | otherwise = [x : ys | x:zxs <- tails' xs, ys <- combinations (k - 1) zxs]

tails' :: [a] -> [[a]]
tails' [] = [[]]
tails' xs = xs : tails' (tail xs)
```

用 `tails'` 枚举每个元素作为组合的第一个元素，递归取剩下 $k-1$ 个。

#### 方法二详解

核心是这一句：

```haskell
[x : ys | x:zxs <- tails' xs, ys <- combinations (k - 1) zxs]
```

**1. `tails'` 做什么**

```haskell
tails' "abc"
-- ["abc", "bc", "c", []]
```

从每个位置截取「还剩的后缀」。最后一个 `[]` 会被后面的模式匹配丢掉。

**2. 模式 `x:zxs <- tails' xs`**

对每个非空后缀，拆成：
- `x` = 这个后缀的**第一个元素**
- `zxs` = 它后面的部分

对 `"abc"`：

| 后缀 | `x` | `zxs` |
|------|-----|-------|
| `"abc"` | `'a'` | `"bc"` |
| `"bc"` | `'b'` | `"c"` |
| `"c"` | `'c'` | `""` |

也就是说：`x` 依次当「当前组合的第一个元素」，且后面只能从 `zxs` 里选——这样就**不会重复、也不会打乱相对顺序**。

**3. 整句在干什么**

固定 `x` 当头后，从 `zxs` 里再选 $k-1$ 个：

```haskell
ys <- combinations (k - 1) zxs
```

得到 `ys`，再拼上 `x : ys`。

**4. 例子：`combinations 2 "abc"`**

```
k=2, xs="abc"

从 "abc": x='a', zxs="bc"
  combinations 1 "bc" = ["b","c"]
  → "ab", "ac"

从 "bc":  x='b', zxs="c"
  combinations 1 "c"  = ["c"]
  → "bc"

从 "c":   x='c', zxs=""
  combinations 1 ""   = []
  → 无

结果: ["ab","ac","bc"]
```

**5. 和「选 / 不选」写法的关系**

| 写法 | 思路 |
|------|------|
| 方法一 | 对首元素：不选 / 选 |
| 方法二 | 枚举「谁当组合的第一个元素」，再从它后面选剩下的 |

方法二用 `tails'` 保证：一旦选了 `x`，后面只能从更右边的元素里选，所以每个组合只出现一次。

**6. 边界**

- `combinations 0 _ = [[]]`：选 0 个，只有一个空组合
- `k < 0`：不可能，返回 `[]`
- `tails'` 到 `[]` 时，`x:zxs` 匹配失败，循环自然结束

### 方法三：使用 subsequences（标准库）

```haskell
import Data.List (subsequences)

combinations :: Int -> [a] -> [[a]]
combinations k xs = filter (\ys -> length ys == k) (subsequences xs)
```

`subsequences` 生成所有子序列（子集），然后只保留长度为 $k$ 的。代码极简，但效率低——生成了 $2^n$ 个候选再过滤，K 接近 n/2 时尤其浪费。

### 方法四：C++ 对照实现

把方法一的递归结构直接翻译成 C++：

```cpp
vector<vector<int>> combinations(int k, const vector<int>& xs) {
    if (k == 0) return {{}};          // combinations 0 _ = [[]]
    if (k < 0)  return {};            // combinations k _ | k < 0 = []
    if (xs.empty()) return {};        // combinations _ [] = []

    // 不选 xs[0]
    auto rest = vector<int>(xs.begin() + 1, xs.end());
    auto without = combinations(k, rest);

    // 选 xs[0]
    auto with = combinations(k - 1, rest);
    for (auto& c : with)
        c.insert(c.begin(), xs[0]);   // map (x :)

    without.insert(without.end(), with.begin(), with.end());
    return without;
}
```

| Haskell | C++ |
|---------|-----|
| `[[]]` | `{{}}` |
| `[]` | `{}` |
| `combinations k xs` | 不选当前元素 |
| `map (x :) (combinations (k-1) xs)` | 选当前元素，插到每个组合前面 |

更常见的迭代写法（DFS 回溯）：

```cpp
vector<vector<int>> combinations(int k, const vector<int>& xs) {
    vector<vector<int>> res;
    vector<int> path;
    function<void(int)> dfs = [&](int start) {
        if ((int)path.size() == k) {
            res.push_back(path);
            return;
        }
        for (int i = start; i < (int)xs.size(); i++) {
            path.push_back(xs[i]);
            dfs(i + 1);
            path.pop_back();
        }
    };
    dfs(0);
    return res;
}
```

## 方法对比

| 方法 | 效率 | 特点 |
|------|------|------|
| 递归回溯 | O(C(n,k))，只生成需要的组合 | 标准写法，推荐 |
| 列表推导 + tails | O(C(n,k)) | 另一种回溯思路 |
| subsequences 过滤 | O(2ⁿ)，生成了大量多余组合 | 代码短，适合小规模 |
| C++ 递归 / DFS | O(C(n,k)) | 对照理解，竞赛常用 |
