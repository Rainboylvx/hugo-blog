# P84 - 用 Prim 算法构造最小生成树

> Construct a minimum spanning tree
>
> 官方模块：`Problems.P84`　核心函数：`minimumSpanningTree`

## 算法

从任意顶点开始，反复选择连接“树内”和“树外”的最小权边。加入这条边不会产生环；连通图在加入 $n-1$ 条边后覆盖全部顶点。

## 实现

### 方法一：Prim 贪心

```haskell
minimumSpanningTree :: G -> Map.Map Edge Int -> G
minimumSpanningTree graph weights
  | Set.null vs = graph
  | otherwise   = grow (Set.singleton (Set.findMin vs)) []
  where
    (vs,es) = canonical graph
    grow inside chosen
      | inside == vs = toG (Lists (Set.toList vs, chosen))
      | null crossing = toG (Lists ([],[]))
      | otherwise =
          let edge@(u,v) = minimumByWeight crossing
              newVertex = if Set.member u inside then v else u
          in grow (Set.insert newVertex inside) (edge : chosen)
      where
        crossing = [e | e@(u,v) <- Set.toList es
                       , Set.member u inside /= Set.member v inside]

    minimumByWeight = minimumBy
      (comparing (\edge -> Map.findWithDefault 0 edge weights))
```

需要导入 `Data.List (minimumBy)` 和 `Data.Ord (comparing)`。权重映射的键必须使用 P80 的规范化无向边。

### 方法二：Kruskal 算法

```haskell
import Data.List (sortOn)

minimumSpanningTreeKruskal :: G -> Map.Map Edge Int -> G
minimumSpanningTreeKruskal graph weights =
  choose sortedEdges initialComponents []
  where
    (vertices, edges) = canonical graph
    sortedEdges = sortOn edgeWeight (Set.toList edges)
    edgeWeight edge = Map.findWithDefault 0 edge weights
    initialComponents = map Set.singleton (Set.toList vertices)

    choose [] _ chosen
      | length chosen == Set.size vertices - 1 = build chosen
      | Set.null vertices = build []
      | otherwise = toG (Lists ([], []))
    choose (edge@(u, v):rest) components chosen
      | componentU == componentV = choose rest components chosen
      | otherwise = choose rest merged (edge : chosen)
      where
        componentU = componentOf u components
        componentV = componentOf v components
        merged = Set.union componentU componentV
               : filter (\component -> component /= componentU && component /= componentV)
                        components

    componentOf vertex =
      head . filter (Set.member vertex)

    build chosen = toG (Lists (Set.toList vertices, chosen))
```

Kruskal 按权重从小到大检查边。这里用顶点集合列表表示连通分量：端点已经属于同一分量时跳过该边，否则合并两个分量。它比并查集版本短，但每次查找和合并分量都是线性的。

## 方法对比

| 方法 | 生长方式 | 当前实现复杂度 |
|------|----------|----------------|
| Prim | 从一个顶点向外扩展 | O(nm) |
| Kruskal | 按全局边权合并分量 | O(m log m + mn) |

## 测试

```haskell
>>> let g = toG (Paths [[1,2,3],[1,3]])
>>> let w = Map.fromList [((1,2),5),((2,3),1),((1,3),2)]
>>> canonical (minimumSpanningTree g w)
(fromList [1,2,3],fromList [(1,3),(2,3)])
>>> canonical (minimumSpanningTreeKruskal g w)
(fromList [1,2,3],fromList [(1,3),(2,3)])
```

当前版本每轮线性扫描所有边，复杂度 $O(nm)$；使用优先队列可优化到 $O(m\log n)$。
