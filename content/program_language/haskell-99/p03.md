# P03 - 取列表第 k 个元素

> Indexed element in a list
>
> 官方模块: `Problems.P03`
> 核心函数: `elementAt`

## 题目描述

找出列表的第 k 个元素（1-based indexing，第一个元素编号为 1）。如果 k 超出列表范围，返回 `Nothing`。

## 函数签名

```haskell
elementAt :: [a] -> Int -> Maybe a
```

## 思路

这是一个带下标的递归：递归遍历列表的同时递减 k，当 k 减到 1 时返回当前元素。

三种终止条件：
- `[]`——列表提前耗尽，返回 `Nothing`
- `k < 1`——非法下标，返回 `Nothing`
- `k == 1`——命中目标，返回 `Just x`

## 实现

### 方法一：递归递减

```haskell
elementAt :: [a] -> Int -> Maybe a
elementAt []      _ = Nothing
elementAt _       k | k < 1 = Nothing
elementAt (x:_)   1 = Just x
elementAt (_:xs)  k = elementAt xs (k - 1)
```

用 guard `k < 1` 处理非法下标，模式 `(x:_)` 配合 `1` 命中目标，`(_:xs)` 配合 `k` 递归缩小。

### 方法二：使用 `drop` + `listToMaybe`

```haskell
import Data.Maybe (listToMaybe)

elementAt :: [a] -> Int -> Maybe a
elementAt xs k
  | k < 1     = Nothing
  | otherwise = listToMaybe (drop (k - 1) xs)
```

`drop (k-1) xs` 丢掉前 k-1 个元素（如果列表不够长，得到空列表），`listToMaybe` 取结果列表的第一个元素。

这种写法的好处是不需要手写递归，语义清晰。

### 方法三：使用 `zip` 过滤

```haskell
elementAt :: [a] -> Int -> Maybe a
elementAt xs k = lookup k (zip [1..] xs)
```

`zip [1..] xs` 将列表标号为 `[(1, x1), (2, x2), ...]`，`lookup k` 按 key 查找。简洁优雅，但效率是 O(n)。

## 测试

```haskell
>>> elementAt [1,2,3] 2
Just 2

>>> elementAt "haskell" 5
Just 'e'

>>> elementAt [1,2] 3
Nothing

>>> elementAt [] 1
Nothing

>>> elementAt [10,20,30] 0
Nothing
```
