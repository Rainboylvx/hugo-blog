# P51 - 纠错码

> Error correction codes
>
> 官方模块：`Problems.P51`　核心函数：`errorCorrectingEncode`, `errorCorrectingDecode`

---

[← P50 霍夫曼编码](./p50) | [P52 合取范式 →](./p52)

---

## 题目描述

用长度为 3 的重复码编码布尔位：`False` 变为三个 `False`，`True` 同理。解码时对每组三位多数表决，因此每组最多一个错误仍可恢复。

## 函数签名

```haskell
errorCorrectingEncode :: [Bool] -> [Bool]
errorCorrectingDecode :: [Bool] -> [Bool]
```

## 实现

### 方法一：replicate + 递归解码

```haskell
errorCorrectingEncode :: [Bool] -> [Bool]
errorCorrectingEncode = concatMap (replicate 3)

errorCorrectingDecode :: [Bool] -> [Bool]
errorCorrectingDecode [] = []
errorCorrectingDecode xs
  | length chunk == 3 = majority chunk : errorCorrectingDecode rest
  | otherwise         = error "incomplete repetition-code block"
  where
    (chunk, rest) = splitAt 3 xs
    majority bits = length (filter id bits) * 2 > length bits
```

编码使用 `concatMap` 表达“每个输入位展开成三位”，解码则按三位分组计数。不是完整码字的尾部会被拒绝，避免悄悄生成没有多数票的结果。

### 方法二：模式匹配 + 多数门

```haskell
errorCorrectingEncode :: [Bool] -> [Bool]
errorCorrectingEncode [] = []
errorCorrectingEncode (bit:bits) =
  bit : bit : bit : errorCorrectingEncode bits

errorCorrectingDecode :: [Bool] -> [Bool]
errorCorrectingDecode [] = []
errorCorrectingDecode (a:b:c:rest) =
  vote a b c : errorCorrectingDecode rest
errorCorrectingDecode _ = error "incomplete repetition-code block"

vote :: Bool -> Bool -> Bool -> Bool
vote a b c = (a && b) || (a && c) || (b && c)
```

这个版本直接匹配三个布尔值，用多数门表达式投票，不构造临时分组，也不计算列表长度。

## 方法对比

| 方法 | 特点 |
|------|------|
| `replicate` + `splitAt` | 列表组合风格，容易推广到其他重复次数 |
| 模式匹配 + 多数门 | 固定三重复码，单次扫描且不构造临时列表 |

## 测试

```haskell
>>> errorCorrectingEncode [True, False]
[True,True,True,False,False,False]
>>> errorCorrectingDecode [True,True,True,False,False,False]
[True,False]
-- 模拟单 bit 错误：
>>> errorCorrectingDecode [False,True,True,True,False,False]
[True,False]
>>> errorCorrectingDecode [True,False]
*** Exception: incomplete repetition-code block
```
