# P64 - 二叉树布局：中序排列

> Binary tree layout: in-order
>
> 官方模块：`Problems.P64`　核心函数：`layoutInorder`

## 题目描述

按中序遍历顺序给二叉树节点分配坐标 $(x, y)$。$x$ 为中序遍历序号（即当前是第几个被访问的节点），$y$ 为层数。

## 函数签名

```haskell
layoutInorder :: Tree a -> Tree (a, (Int, Int))
```

## 实现

### 方法一：递归分配坐标

```haskell
layoutInorder :: Tree a -> Tree (a, (Int, Int))
layoutInorder tree = fst (layout tree 1 1)

layout :: Tree a -> Int -> Int -> (Tree (a, (Int, Int)), Int)
layout Empty next _ = (Empty, next)
layout (Branch x left right) next y =
  let (leftTree, next')   = layout left next (y + 1)
      (rightTree, next'') = layout right (next' + 1) (y + 1)
  in (Branch (x, (next', y)) leftTree rightTree, next'')
```

中序遍历：先布局左子树得到 `next'`，当前节点 x = `next'`，再布局右子树，起始序号为 `next' + 1`。

### 方法二：利用左子树大小定位

```haskell
layoutInorder :: Tree a -> Tree (a, (Int, Int))
layoutInorder tree = place tree 0 1
  where
    place Empty _ _ = Empty
    place (Branch value left right) skipped depth =
      Branch (value, (rootX, depth))
        (place left skipped (depth + 1))
        (place right rootX (depth + 1))
      where
        rootX = skipped + treeSize left + 1
```

先由 `treeSize left` 知道左子树会占用多少个中序位置。若此前已经跳过 `skipped` 个节点，根的位置就是 `skipped + treeSize left + 1`。它用子树大小直接定位，不需要在线程中返回下一个序号。

## 方法对比

| 方法 | 状态来源 | 复杂度 |
|------|----------|--------|
| 递归传递下一个序号 | 左子树布局后的返回值 | O(n) |
| 利用左子树大小 | 每个节点重新计算左子树大小 | 朴素实现最坏 O(n²) |

## 测试

```haskell
>>> layoutInorder (leaf 'a')
Branch ('a',(1,1)) Empty Empty
>>> layoutInorder (Branch 'a' (leaf 'b') (leaf 'c'))
Branch ('a',(2,1)) (Branch ('b',(1,2)) Empty Empty) (Branch ('c',(3,2)) Empty Empty)
```
