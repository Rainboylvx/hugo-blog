# P53 - 用归结原理证明命题

> The resolution rule
>
> 官方模块：`Problems.P53`　核心函数：`isTheorem`

---

## 原理

要证明公理集合 $A$ 蕴含结论 $C$，把 $A\land\neg C$ 转成 CNF，然后尝试推出空子句。若推出矛盾，原命题成立。

## 实现

```haskell
import qualified Data.Set as Set

type Clause = Set.Set Formula

isTheorem :: [Formula] -> Formula -> Bool
isTheorem axioms conjecture = refute initial
  where
    Conjoin clauses = toConjunctiveNormalForm
      (Conjoin (Complement conjecture : axioms))
    initial = Set.fromList
      [Set.fromList literals | Disjoin literals <- clauses]

refute :: Set.Set Clause -> Bool
refute known
  | Set.member Set.empty known = True
  | Set.null fresh             = False
  | otherwise                  = refute (Set.union known fresh)
  where
    clauses = Set.toList known
    generated = Set.fromList
      [r | a <- clauses, b <- clauses, r <- resolve a b, not (tautology r)]
    fresh = Set.difference generated known

resolve :: Clause -> Clause -> [Clause]
resolve a b =
  [ Set.union (Set.delete literal a) (Set.delete opposite b)
  | literal <- Set.toList a
  , let opposite = complementLiteral literal
  , Set.member opposite b
  ]

complementLiteral :: Formula -> Formula
complementLiteral (Complement f) = f
complementLiteral f              = Complement f

tautology :: Clause -> Bool
tautology clause = any
  (\literal -> Set.member (complementLiteral literal) clause)
  clause
```

集合既去重，也让“是否已经生成过这个子句”的判断明确。每轮至少加入一个新子句，否则停止。

## 测试

```haskell
>>> let p = Variable "p"; q = Variable "q"
>>> isTheorem [p, Disjoin [Complement p, q]] q
True
>>> isTheorem [p] q
False
>>> isTheorem [] (Disjoin [p, Complement p])
True
```

## 复杂度

归结在最坏情况下会生成指数数量的子句。这是命题逻辑判定本身的复杂性，不是集合实现造成的偶然开销。
