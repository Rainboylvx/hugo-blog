## 题目描述

每行、每列给出连续实心块的长度。相邻块之间至少有一个空格；求满足全部行列线索的布尔矩阵。

## 生成单行候选

```haskell
import Data.List (isPrefixOf, transpose)

linePatterns :: Int -> [Int] -> [[Bool]]
linePatterns width [] = [replicate width False]
linePatterns width (block:rest)
  | block <= 0 = []
  | otherwise =
      [ replicate start False ++ replicate block True ++ separator ++ suffix
      | start <- [0..width - block - minimumTail]
      , let separator = if null rest then [] else [False]
      , suffix <- linePatterns (width - start - block - length separator) rest]
  where
    minimumTail = sum rest + length rest
```

`minimumTail` 是剩余块及它们所需分隔空格的最小长度，因此起点枚举不会产生越界布局。

## 求解

### 方法一：按行回溯并检查列前缀

```haskell
nonogram :: [[Int]] -> [[Int]] -> Maybe [[Bool]]
nonogram rowClues columnClues = build [] rowOptions
  where
    height = length rowClues
    width  = length columnClues
    rowOptions = map (linePatterns width) rowClues
    columnOptions = map (linePatterns height) columnClues

    build rows []
      | transpose rows `matches` columnOptions = Just rows
      | otherwise = Nothing
    build rows (options:remaining) = firstJust
      [build rows' remaining | row <- options
                             , let rows' = rows ++ [row]
                             , compatible rows']

    compatible rows = and
      [any (column `isPrefixOf`) options
      | (column, options) <- zip (transpose rows) columnOptions]

    actual `matches` options = and (zipWith elem actual options)

firstJust [] = Nothing
firstJust (Just value:_) = Just value
firstJust (Nothing:rest) = firstJust rest
```

### 方法二：从候选较少的方向搜索

```haskell
nonogramAdaptive :: [[Int]] -> [[Int]] -> Maybe [[Bool]]
nonogramAdaptive rowClues columnClues
  | rowCost <= columnCost = nonogram rowClues columnClues
  | otherwise = transpose <$> nonogram columnClues rowClues
  where
    height = length rowClues
    width = length columnClues
    rowCost = product
      [toInteger (length (linePatterns width clue)) | clue <- rowClues]
    columnCost = product
      [toInteger (length (linePatterns height clue)) | clue <- columnClues]
```

横纵转置后仍是同一个 Nonogram。先估算逐行和逐列候选组合数，从更小的一侧调用基准求解器；按列求解得到的棋盘最后再转置回来。线索不对称时，这个选择可以显著减少搜索分支。

## 方法对比

| 方法 | 搜索方向 |
|------|----------|
| 固定按行 | 实现直接，表现取决于行线索强弱 |
| 自适应 | 比较候选组合数后选择行或列 |

## 测试

```haskell
>>> fmap (map (map (\b -> if b then '#' else '.')))
... (nonogram [[1],[3],[1]] [[1],[3],[1]])
Just [".#.","###",".#."]
>>> nonogram [[3]] [[1],[1]]
Nothing
>>> nonogramAdaptive [[1],[3],[1]] [[1],[3],[1]] ==
... nonogram [[1],[3],[1]] [[1],[3],[1]]
True
```

逐行加入后立即检查所有列前缀，可以在错误行布局出现时马上回溯。
