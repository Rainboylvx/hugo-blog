# P37 - 欧拉乘积公式求 φ(n)

> Euler's totient function with Euler's product formula
>
> 官方模块：`Problems.P37`　核心函数：`totient'`

---

[← P36 带重数的质因数分解](./p36) | [P38 高欧拉函数数 →](./p38)

---

## 核心公式

若 $m=\prod_i p_i^{k_i}$，则

$$
\varphi(m)=\prod_i p_i^{k_i-1}(p_i-1).
$$

## 函数签名

```haskell
totient' :: Integral a => a -> a
```

## 实现

### 方法一：基于 P36 的质因数分解

```haskell
totient' :: Integral a => a -> a
totient' m
  | m < 1     = error "totient': positive input required"
  | otherwise = product
      [p ^ (k - 1) * (p - 1) | (p, k) <- primeFactorsMultiplicity m]
```

P36 已经给出所有 $(p_i,k_i)$，这里只需逐项映射并相乘。空乘积等于 1，所以 `totient' 1` 会自然得到 1。

### 方法二：不用 P36，只用 P35 的质因数列表

```haskell
import Data.List (group)

totient' :: Integral a => a -> a
totient' m
  | m < 1     = error "totient': positive input required"
  | otherwise = foldl apply m (group (primeFactors m))
  where
    apply n xs = n `div` p * (p - 1)
      where p = head xs
```

从 $\varphi(m) = m\prod_{p|m}(1-1/p)$ 出发，每遇到一个不同质因数，做 `n / p * (p-1)`。不依赖 `primeFactorsMultiplicity`，只依赖 `primeFactors`。

### 方法三：与 P34 对比的定义法

```haskell
totient' :: Integral a => a -> a
totient' m = totient m   -- 复用 P34 的定义实现
```

如果要对比性能，可以直接调用 P34。对小数字结果相同，对大数字 P34 慢得多。

## 方法对比

| 方法 | 复杂度 | 依赖 | 特点 |
|------|--------|------|------|
| 乘积公式 + P36 | O(√m) | P36 | 标准做法，推荐 |
| 逐个质因数约化 | O(√m) | P35 | 不需要重数信息 |
| P34 定义法 | O(m log m) | P34 | 仅用于验证一致性 |

## 测试

```haskell
>>> totient' 10
4
>>> totient' 315
144
>>> all (\n -> totient n == totient' n) [1..100]
True
```
