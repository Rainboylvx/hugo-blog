# P28 - 按子列表长度及其频率排序

> Sort a list of lists according to sublist length
>
> 官方模块：`Problems.P28`　核心函数：`lsort`、`lfsort`

## 题目描述

实现两种排序：

- `lsort` 按子列表长度从短到长排序；
- `lfsort` 按“该长度在输入中出现的次数”排序，稀有长度优先。

## 函数签名

```haskell
lsort  :: [[a]] -> [[a]]
lfsort :: [[a]] -> [[a]]
```

## 实现

`sortOn` 先计算键，再根据键稳定排序。`lfsort` 先建立“长度到出现次数”的映射。

```haskell
import Data.List (sortOn)
import qualified Data.Map.Strict as Map

lsort :: [[a]] -> [[a]]
lsort = sortOn length

lfsort :: [[a]] -> [[a]]
lfsort xs = sortOn frequency xs
  where
    counts = Map.fromListWith (+)
      [(length ys, 1 :: Int) | ys <- xs]
    frequency ys = Map.findWithDefault 0 (length ys) counts
```

如果长度 4 只出现一次、长度 3 出现两次、长度 2 出现三次，那么 `lfsort` 会依次放置长度 4、3、2 的列表。

## 测试

```haskell
>>> lsort ["xxx","xx","xxx","xx","xxxx","xx","x"]
["x","xx","xx","xx","xxx","xxx","xxxx"]
>>> lfsort ["xxx","xx","xxx","xx","xxxx","xx"]
["xxxx","xxx","xxx","xx","xx","xx"]
>>> lsort []
[]
```

## 复杂度

设有 $n$ 个子列表、元素总数为 $L$。计算长度共 $O(L)$，排序为 $O(n\log n)$，映射统计还需要 $O(n\log n)$。

## 要点总结

- `sortOn` 适合“先投影出排序键”的问题。
- `lfsort` 的键不是列表长度，而是这个长度的全局频率。
- Haskell 的 `sortOn` 是稳定排序，频率相同时保留原相对顺序。
