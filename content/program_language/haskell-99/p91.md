# P91 - 骑士巡游

> Knight's tour
>
> 官方模块：`Problems.P91`　核心函数：`knightsTour`、`closedKnightsTour`

## Warnsdorff 规则

回溯时优先尝试“下一步可走位置最少”的格子，让最受约束的位置尽早决定，显著减少死路搜索。

## 实现

### 方法一：Warnsdorff 排序回溯

```haskell
import Data.List (sort, sortOn)
import qualified Data.Set as Set

type Position = (Int, Int)

knightsTour :: Int -> Position -> Maybe [Position]
knightsTour n end
  | not (inside n end) = Nothing
  | otherwise = search n [end] (Set.delete end board)
  where board = Set.fromList [(x,y) | x <- [1..n], y <- [1..n]]

search n path@(current:_) remaining
  | Set.null remaining = Just path
  | otherwise = firstJust
      [search n (next:path) (Set.delete next remaining)
      | next <- ranked]
  where
    ranked = sortOn (length . moves remaining) (moves remaining current)
search _ [] _ = Nothing

moves remaining (x,y) =
  filter (`Set.member` remaining)
    [(x+dx,y+dy) | (dx,dy) <- jumps]
  where jumps = [(1,2),(1,-2),(-1,2),(-1,-2),
                 (2,1),(2,-1),(-2,1),(-2,-1)]

firstJust [] = Nothing
firstJust (Just x:_) = Just x
firstJust (Nothing:xs) = firstJust xs

inside n (x,y) = 1 <= x && x <= n && 1 <= y && y <= n
```

从终点反向搜索，最终得到的列表自然以指定位置结尾。

## 闭合巡游

```haskell
closedKnightsTour :: Int -> Maybe [Position]
closedKnightsTour n
  | n < 1     = Nothing
  | otherwise = firstJust
      [pathTo end [(1,1)] (Set.delete end withoutStart)
      | end <- moves withoutStart (1,1)]
  where
    board = Set.fromList [(x,y) | x <- [1..n], y <- [1..n]]
    withoutStart = Set.delete (1,1) board

    pathTo end path@(current:_) remaining
      | Set.null remaining =
          if knightMove current end
            then Just (reverse path ++ [end])
            else Nothing
      | otherwise = firstJust
          [pathTo end (next:path) (Set.delete next remaining)
          | next <- sortOn (length . moves remaining)
                           (moves remaining current)]
    pathTo _ [] _ = Nothing

    knightMove (x,y) (x',y') =
      sort [abs (x-x'), abs (y-y')] == [1,2]
```

### 方法二：朴素回溯

```haskell
knightsTourPlain :: Int -> Position -> Maybe [Position]
knightsTourPlain n end
  | not (inside n end) = Nothing
  | otherwise = searchPlain [end] (Set.delete end board)
  where
    board = Set.fromList [(x, y) | x <- [1..n], y <- [1..n]]

    searchPlain path@(current:_) remaining
      | Set.null remaining = Just path
      | otherwise = firstJust
          [searchPlain (next : path) (Set.delete next remaining)
          | next <- moves remaining current]
    searchPlain [] _ = Nothing
```

朴素版本保持相同的回溯状态，却完全按照坐标顺序尝试下一步。它适合说明 Warnsdorff 排序没有改变解集合，只是把最可能形成死路的选择提前。

## 方法对比

| 方法 | 候选顺序 | 实际表现 |
|------|----------|----------|
| Warnsdorff | 后续出口少的格子优先 | 大棋盘通常很快 |
| 朴素回溯 | 坐标顺序 | 搜索树巨大，只适合小棋盘对照 |

## 测试

```haskell
>>> fmap length (knightsTour 6 (3,5))
Just 36
>>> fmap length (closedKnightsTour 6)
Just 36
>>> knightsTour 3 (1,1)
Nothing
>>> knightsTourPlain 1 (1,1)
Just [(1,1)]
```
