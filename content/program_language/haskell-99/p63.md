# P63 - 构造并判断完全二叉树

> Construct and recognize complete binary trees
>
> 官方模块：`Problems.P63`　核心函数：`completeBinaryTree`, `isCompleteBinaryTree`

---

## 题目描述

完全二叉树中，除最后一层外每层都被填满，且最后一层的节点靠左排列。

## 函数签名

```haskell
completeBinaryTree :: Int -> Tree ()
isCompleteBinaryTree :: Tree a -> Bool
```

## 数组下标模型

把根编号为 1，则编号 $i$ 的左右孩子分别是 $2i$ 和 $2i+1$。完全二叉树恰好占用连续编号 $1..n$。

## 实现

### 方法一：递归下标构造 + 验证

```haskell
completeBinaryTree :: Int -> Tree ()
completeBinaryTree size = build 1
  where
    build i
      | i > size  = Empty
      | otherwise = Branch () (build (2*i)) (build (2*i + 1))

isCompleteBinaryTree :: Tree a -> Bool
isCompleteBinaryTree tree = valid tree 1
  where
    size = treeSize tree
    valid Empty i = i > size
    valid (Branch _ left right) i =
      i <= size && valid left (2*i) && valid right (2*i + 1)
```

### 方法二：isComplete 用 BFS

```haskell
isCompleteBinaryTree :: Tree a -> Bool
isCompleteBinaryTree tree = scan [tree] False
  where
    scan [] _ = True
    scan (Empty : rest) _ = scan rest True
    scan (Branch _ _ _ : _) True = False
    scan (Branch _ left right : rest) False =
      scan (rest ++ [left, right]) False
```

BFS 层序遍历时，一旦遇到 `Empty`，之后就不能再有非空节点。

### 方法三：按左右子树规模构造

```haskell
completeBinaryTree :: Int -> Tree ()
completeBinaryTree n
  | n <= 0    = Empty
  | otherwise = Branch ()
      (completeBinaryTree leftSize)
      (completeBinaryTree (n - 1 - leftSize))
  where
    height = floorLog2 n
    lastLevel = n - (2 ^ height - 1)
    leftCapacity = 2 ^ max 0 (height - 1)
    leftSize
      | height == 0 = 0
      | otherwise = leftCapacity - 1 + min lastLevel leftCapacity

floorLog2 :: Int -> Int
floorLog2 value = length (takeWhile (<= value) (iterate (* 2) 1)) - 1
```

高度确定后，最后一层优先填入左子树。这个版本直接算出左右子树各有多少节点，不使用堆下标。

## 方法对比

| 任务 | 方法一 | 替代方法 |
|------|--------|----------|
| 构造 | 堆下标是否超过 n | 计算左右子树规模 |
| 判断 | 最大下标与节点数比较 | BFS 检查空位后的节点 |

## 测试

```haskell
>>> treeSize (completeBinaryTree 6)
6
>>> isCompleteBinaryTree (completeBinaryTree 6)
True
>>> isCompleteBinaryTree (Branch 1 Empty (leaf 2))
False
```
