# P35 - 质因数分解

> Determine the prime factors of a positive integer
>
> 官方模块：`Problems.P35`　核心函数：`primeFactors`

---

[← P34 欧拉函数](./p34) | [P36 带重数的质因数分解 →](./p36)

---

## 题目描述

把正整数分解为按升序排列的质因数列表，重复因子要重复出现。例如 $315=3\times3\times5\times7$。

## 函数签名

```haskell
primeFactors :: Integral a => a -> [a]
```

## 实现

### 方法一：试除递归（2, 3, 5, 7, ...）

```haskell
primeFactors :: Integral a => a -> [a]
primeFactors n
  | n < 1     = error "primeFactors: positive input required"
  | otherwise = factor n 2
  where
    factor 1 _ = []
    factor x d
      | d * d > x      = [x]
      | x `mod` d == 0 = d : factor (x `div` d) d
      | d == 2         = factor x 3
      | otherwise      = factor x (d + 2)
```

找到因子后仍用同一个 `d` 继续除，从而保留重数。若 $d^2>x$，剩余的 $x$ 必然是质数。

### 方法二：基于 P39 的素数列表

```haskell
primeFactors :: Integral a => a -> [a]
primeFactors n
  | n < 1     = error "primeFactors: positive input required"
  | otherwise = factor n primes
  where
    factor 1 _ = []
    factor x (p:ps)
      | p * p > x   = [x]
      | x `mod` p == 0 = p : factor (x `div` p) (p:ps)
      | otherwise   = factor x ps
```

只试除已知的素数，而不是全部奇数。对于大质因数的情况，比方法一节省约一半的试除次数。

### 方法三：迭代式（不使用递归）

```haskell
import Data.List (unfoldr)

primeFactors :: Integral a => a -> [a]
primeFactors n
  | n < 1     = error "primeFactors: positive input required"
  | otherwise = unfoldr factor n
  where
    factor 1 = Nothing
    factor x = Just (p, x `div` p)
      where p = head [d | d <- [2..], d * d > x || x `mod` d == 0]
```

用 `unfoldr` 生成器风格表达分解过程。每步找到最小的质因数，输出它，继续分解剩余部分。代码更函数式但效率不如方法一。

## 方法对比

| 方法 | 试除对象 | 特点 |
|------|----------|------|
| 试除奇数 | 2, 3, 5, 7... | 自包含，无外部依赖 |
| 素数列表 | P39 的 `primes` | 试除次数最少 |
| unfoldr 迭代 | 动态枚举 | 函数式风格，代码最简洁 |

## 测试

```haskell
>>> primeFactors 315
[3,3,5,7]
>>> primeFactors 13
[13]
>>> primeFactors 1
[]
```
