# P12 - 解码游程编码

> Decode a run-length encoded list
>
> 官方模块: `Problems.P12`
> 核心函数: `decodeModified`

## 题目描述

将 P11 生成的 `Encoding` 编码列表解码，还原为原始列表。`Single x` 展开为单个 `x`，`Multiple n x` 展开为 n 个 `x`。

## 函数签名

```haskell
decodeModified :: [Encoding a] -> [a]
```

## 思路

遍历编码列表，对每个编码：
- `Single x` → `[x]`
- `Multiple n x` → `replicate n x`

然后用 `concat` 把所有结果拼接起来。

## 实现

### 方法一：concatMap

```haskell
decodeModified :: [Encoding a] -> [a]
decodeModified = concatMap expand
  where
    expand (Single x)     = [x]
    expand (Multiple n x) = replicate n x
```

最直接的实现，`concatMap` 遍历并展平。

### 方法二：列表推导

```haskell
decodeModified :: [Encoding a] -> [a]
decodeModified [] = []
decodeModified (Single x : rest)     = x : decodeModified rest
decodeModified (Multiple n x : rest) = replicate n x ++ decodeModified rest
```

显式递归，逐个处理编码。

### 方法三：使用 `foldr`

```haskell
decodeModified :: [Encoding a] -> [a]
decodeModified = foldr expand []
  where
    expand (Single x)     acc = x : acc
    expand (Multiple n x) acc = replicate n x ++ acc
```

## 为什么叫"改进的"编码？

对比 P10 的标准编码 `[(Int, a)]` 和 P11 的改进编码 `[Encoding a]`：

```
P10:   [(4,'a'), (1,'b'), (2,'c'), (2,'a'), (1,'d'), (4,'e')]
P11:   [Multiple 4 'a', Single 'b', Multiple 2 'c', Multiple 2 'a', Single 'd', Multiple 4 'e']
```

改进之处在于**用类型系统表达语义**——`Single` 明确表示"只有一个"，比 `(1, 'a')` 更自解释。

## 测试

```haskell
>>> decodeModified [Multiple 4 'a', Single 'b', Multiple 2 'c', Multiple 2 'a', Single 'd', Multiple 4 'e']
"aaaabccaadeeee"

>>> decodeModified []
[]

>>> decodeModified [Single 'x']
"x"
```
