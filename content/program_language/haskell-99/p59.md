# P59 - 构造高度平衡二叉树

> Construct height-balanced binary trees
>
> 官方模块：`Problems.P59`　核心函数：`heightBalancedTrees`

---

[← P58 对称平衡树](./p58) | [P60 指定节点数的高度平衡树 →](./p60)

---

## 题目描述

高度平衡要求每个节点的左右子树高度差不超过 1。给定高度 $h$，生成所有恰好为该高度的树形。

## 函数签名

```haskell
heightBalancedTrees :: Int -> [Tree ()]
```

## 实现

### 方法一：组合高度（标准解）

```haskell
heightBalancedTrees :: Int -> [Tree ()]
heightBalancedTrees 0 = [Empty]
heightBalancedTrees 1 = [leaf ()]
heightBalancedTrees h
  | h < 0     = []
  | otherwise =
      combine tall tall ++ combine tall short ++ combine short tall
  where
    tall  = heightBalancedTrees (h - 1)
    short = heightBalancedTrees (h - 2)
    combine ls rs = [Branch () l r | l <- ls, r <- rs]
```

要让根的高度为 $h$，至少一侧必须高 $h-1$；另一侧只能是 $h-1$ 或 $h-2$，所以恰好有三种高度组合。

### 方法二：惰性动态规划表

```haskell
heightBalancedTrees :: Int -> [Tree ()]
heightBalancedTrees h
  | h < 0     = []
  | otherwise = table !! h
  where
    table = [ [Empty], [leaf ()] ] ++ map build [2..]

    build height =
      combine tall tall ++ combine tall short ++ combine short tall
      where
        tall = table !! (height - 1)
        short = table !! (height - 2)

    combine leftTrees rightTrees =
      [Branch () left right | left <- leftTrees, right <- rightTrees]
```

惰性列表保存每个高度的全部结果。高度 $h$ 只引用已经定义的 $h-1$ 和 $h-2$ 项，因此多个高度查询可以复用子问题，而不是重新递归生成。

### 辅助推导：给定高度求最小节点数

```haskell
minNodes :: Int -> Int
minNodes 0 = 0
minNodes 1 = 1
minNodes h = 1 + minNodes (h - 1) + minNodes (h - 2)
```

辅助函数：高度为 h 的高度平衡树至少有 $M_h$ 个节点，$M_h = 1 + M_{h-1} + M_{h-2}$（类斐波那契）。

## 测试

```haskell
>>> length (heightBalancedTrees 3)
15
>>> all ((== 3) . treeHeight) (heightBalancedTrees 3)
True
>>> heightBalancedTrees 0
[Empty]
```
