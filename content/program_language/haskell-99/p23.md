# P23 - 从列表随机选择 N 个元素

> Extract a given number of randomly selected elements from a list
>
> 官方模块：`Problems.P23`　核心函数：`randomSelect`

## 题目描述

从列表中随机选出 $n$ 个不同位置的元素。一个元素一旦选中就从候选列表删除，因此不会重复抽中同一位置。函数还要返回更新后的随机数生成器。

## 函数签名

```haskell
import System.Random (RandomGen)

randomSelect :: RandomGen g => [a] -> Int -> g -> ([a], g)
```

显式传入和返回生成器，能让函数保持纯函数：同一列表、数量和种子总会产生同一结果。

## 实现

每轮生成一个合法下标，用 `removeAt0` 同时取出元素和剩余列表，然后继续抽取。

```haskell
import System.Random (RandomGen, randomR)

randomSelect :: RandomGen g => [a] -> Int -> g -> ([a], g)
randomSelect xs count gen = go xs (max 0 count) gen []
  where
    go [] _ g picked = (reverse picked, g)
    go _  0 g picked = (reverse picked, g)
    go pool k g picked =
      let (i, g')       = randomR (0, length pool - 1) g
          (x, pool')    = removeAt0 i pool
      in go pool' (k - 1) g' (x : picked)

removeAt0 :: Int -> [a] -> (a, [a])
removeAt0 i xs =
  case splitAt i xs of
    (front, x : back) -> (x, front ++ back)
    _ -> error "removeAt0: index out of bounds"
```

这里的错误分支不会从 `randomSelect` 触发，因为 `randomR` 的范围由当前列表长度决定。

## 测试

```haskell
import System.Random (mkStdGen)

>>> length . fst $ randomSelect "abcdefgh" 3 (mkStdGen 111)
3
>>> let (picked, _) = randomSelect [1..5] 10 (mkStdGen 1)
>>> length picked == 5
True
```

> [!IMPORTANT] 随机测试
> 不要把某个版本 `random` 包产生的具体排列当成唯一正确答案。更稳定的性质是：结果长度正确、每个结果都来自输入，并且同一位置不会抽取两次。

## 复杂度

列表的 `length`、`splitAt` 和 `++` 都是线性操作。抽取 $k$ 次的最坏复杂度为 $O(kn)$；若追求性能，应换用可随机访问并高效删除的数据结构。
