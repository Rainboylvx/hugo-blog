# P68 - 根据中序和前序序列重建二叉树

> In-order and pre-order sequences of binary trees
>
> 官方模块：`Problems.P68`　核心函数：`inorder`、`preorder`、`ordersToTree`

## 遍历

```haskell
inorder :: Tree a -> [a]
inorder Empty = []
inorder (Branch x left right) = inorder left ++ [x] ++ inorder right

preorder :: Tree a -> [a]
preorder Empty = []
preorder (Branch x left right) = [x] ++ preorder left ++ preorder right
```

## 重建思路

前序首元素是根。在中序序列中找到根，其左侧长度就是左子树节点数；据此切分剩余前序序列，再递归重建两侧。题目约定节点值互不相同。

### 方法一：列表切分

```haskell
ordersToTree :: Eq a => [a] -> [a] -> Maybe (Tree a)
ordersToTree [] [] = Just Empty
ordersToTree inorderValues (root:preorderValues) = do
  let (leftIn, rest) = break (== root) inorderValues
  rightIn <- case rest of
    _:xs -> Just xs
    []   -> Nothing
  let (leftPre, rightPre) = splitAt (length leftIn) preorderValues
  left  <- ordersToTree leftIn leftPre
  right <- ordersToTree rightIn rightPre
  pure (Branch root left right)
ordersToTree _ _ = Nothing
```

### 方法二：中序位置索引

```haskell
import qualified Data.Map.Strict as Map

ordersToTreeFast :: Ord a => [a] -> [a] -> Maybe (Tree a)
ordersToTreeFast inorderValues preorderValues
  | length inorderValues /= length preorderValues = Nothing
  | Map.size positions /= length inorderValues = Nothing
  | otherwise = do
      (tree, rest) <- build 0 (length inorderValues - 1) preorderValues
      if null rest then Just tree else Nothing
  where
    positions = Map.fromList (zip inorderValues [0..])

    build low high remaining
      | low > high = Just (Empty, remaining)
    build low high (root:rest) = do
      position <- Map.lookup root positions
      if position < low || position > high
        then Nothing
        else do
          (left, afterLeft) <- build low (position - 1) rest
          (right, afterRight) <- build (position + 1) high afterLeft
          pure (Branch root left right, afterRight)
    build _ _ [] = Nothing
```

中序值到位置的 `Map` 让每次根节点定位从线性查找降为 $O(\log n)$，同时用上下界表示子序列，避免重复执行 `break` 和 `splitAt`。

## 方法对比

| 方法 | 根位置查找 | 最坏复杂度 |
|------|------------|------------|
| 列表切分 | `break` 线性扫描 | O(n²) |
| 位置索引 | `Map.lookup` | O(n log n) |

## 测试

```haskell
>>> inorder tree1
"dbeacgf"
>>> preorder tree1
"abdecfg"
>>> ordersToTree (inorder tree1) (preorder tree1) == Just tree1
True
>>> ordersToTree "ab" "a"
Nothing
>>> ordersToTreeFast (inorder tree1) (preorder tree1) == Just tree1
True
```

若把 `Map` 换成适合节点编号的数组，中序位置查找还可以进一步降为 $O(1)$，使整体达到 $O(n)$。
