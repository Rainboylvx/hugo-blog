# P27 - 将元素划分为互斥分组

> Group elements into disjoint subsets
>
> 官方模块：`Problems.P27`　核心函数：`disjointGroups`

---

## 题目描述

给定各组大小和一组互不相同的元素，生成所有互不相交的分组方案。例如把 9 人分成大小为 2、3、4 的三个有序角色组。

## 函数签名

```haskell
disjointGroups :: [Int] -> [a] -> [[[a]]]
```

## 思路

关键不是只生成当前组，而是同时得到“选中的元素”和“剩余元素”。`choose` 返回所有 `(选择, 剩余)`，然后对剩余元素递归处理后续组。

## 实现

```haskell
disjointGroups :: [Int] -> [a] -> [[[a]]]
disjointGroups [] xs
  | null xs   = [[]]
  | otherwise = []
disjointGroups (n:ns) xs =
  [ group : groups
  | (group, rest) <- choose n xs
  , groups <- disjointGroups ns rest
  ]

choose :: Int -> [a] -> [([a], [a])]
choose 0 xs     = [([], xs)]
choose _ []     = []
choose n (x:xs)
  | n < 0 = []
  | otherwise =
      [ (x : picked, rest) | (picked, rest) <- choose (n - 1) xs ]
      ++
      [ (picked, x : rest) | (picked, rest) <- choose n xs ]
```

第一条列表推导选择 `x`，第二条跳过 `x`。两条分支都保留未选元素，所以不同组不会共享成员。

## 测试

```haskell
>>> length (disjointGroups [2,3,4] [1..9])
1260
>>> disjointGroups [1,1] "ab"
[["a","b"],["b","a"]]
>>> disjointGroups [2,2] [1,2,3]
[]
```

注意组本身有顺序：大小相同的两个组互换后仍是不同结果。这与官方定义一致。

## 复杂度

输出规模是多项式系数 $\frac{n!}{n_1!n_2!\cdots n_k!}$，算法至少需要与输出数量成正比的时间。
