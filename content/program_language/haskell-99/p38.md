# P38 - 高欧拉函数数

> Construct the list of highly totient numbers
>
> 官方模块：`Problems.P38`　核心函数：`highlyTotientNumbers`

---

[← P37 欧拉乘积公式](./p37) | [P39 素数列表 →](./p39)

---

## 题目描述

可能有多个 $x$ 满足 $\varphi(x)=n$。若 $n$ 的原像数量严格大于所有更小整数的原像数量，就称 $n$ 为高欧拉函数数（highly totient number）。要求构造它们的无限列表。

## 函数签名

```haskell
highlyTotientNumbers :: [Integer]
```

## 实现

### 方法一：无限惰性列表（标准解）

```haskell
highlyTotientNumbers :: [Integer]
highlyTotientNumbers = records 1 0
  where
    records n best
      | count > best = n : records (n + 1) count
      | otherwise    = records (n + 1) best
      where count = tally n

tally :: Integer -> Int
tally n = length [x | x <- [1..upperBound n], totient' x == n]

upperBound :: Integer -> Integer
upperBound n = product (map (+ 1) (1 : primeFactors n))
```

困难在于不能无限枚举 $x$。若 $n=\varphi(x)$，可由 $n$ 的质因数构造一个安全上界 $x\le \prod_{q\in\{1\}\cup\operatorname{factors}(n)}(q+1)$。因此 `tally n` 只需检查有限区间。

### 方法二：流式记录值筛选

```haskell
highlyTotientNumbers :: [Integer]
highlyTotientNumbers =
  [n
  | (n, count, previousBest) <- zip3 [1..] counts bestsBefore
  , count > previousBest
  ]
  where
    counts = map tally [1..]
    bestsBefore = scanl max 0 counts
```

把每个 $n$ 的原像数量组成惰性流，`scanl max 0 counts` 在同一位置给出此前最大值。当前数量严格超过此前记录时，$n$ 就是新的高欧拉函数数。它把“计算计数”和“寻找记录”拆成两个独立步骤。

### 方法三：有限前缀一次性统计

```haskell
import qualified Data.Map.Strict as Map

highlyTotientNumbersUpTo :: Integer -> [Integer]
highlyTotientNumbersUpTo limit
  | limit < 1 = []
  | otherwise = records 0 [(n, count n) | n <- [1..limit]]
  where
    searchLimit = maximum (map upperBound [1..limit])
    histogram = Map.fromListWith (+)
      [(totient' x, 1 :: Int) | x <- [1..searchLimit]]
    count n = Map.findWithDefault 0 n histogram

    records _ [] = []
    records best ((n, occurrences):rest)
      | occurrences > best = n : records occurrences rest
      | otherwise = records best rest
```

若只需要不超过 `limit` 的前缀，可以先取这些目标值所需安全上界的最大值，再把每个 $x$ 的欧拉函数值一次性加入直方图。与前两种方法对每个 $n$ 重新扫描不同，这里每个候选 $x$ 只计算一次。

## 方法对比

| 方法 | 特点 |
|------|------|
| 递归记录 | 逐个计算并立即输出，控制流直接 |
| 流式筛选 | `scanl` 维护此前最大值，组合性好 |
| 有限直方图 | 一次统计整个前缀，适合已知上限 |

## 测试

```haskell
>>> take 10 highlyTotientNumbers
[1,2,4,8,12,24,48,72,144,240]
>>> highlyTotientNumbersUpTo 12
[1,2,4,8,12]
```
