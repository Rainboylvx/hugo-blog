# P38 - 高欧拉函数数

> Construct the list of highly totient numbers
>
> 官方模块：`Problems.P38`　核心函数：`highlyTotientNumbers`

---

[← P37 欧拉乘积公式](./p37) | [P39 素数列表 →](./p39)

---

## 题目描述

可能有多个 $x$ 满足 $\varphi(x)=n$。若 $n$ 的原像数量严格大于所有更小整数的原像数量，就称 $n$ 为高欧拉函数数（highly totient number）。要求构造它们的无限列表。

## 函数签名

```haskell
highlyTotientNumbers :: [Integer]
```

## 实现

### 方法一：无限惰性列表（标准解）

```haskell
highlyTotientNumbers :: [Integer]
highlyTotientNumbers = records 1 0
  where
    records n best
      | count > best = n : records (n + 1) count
      | otherwise    = records (n + 1) best
      where count = tally n

tally :: Integer -> Int
tally n = length [x | x <- [1..upperBound n], totient' x == n]

upperBound :: Integer -> Integer
upperBound n = product (map (+ 1) (1 : primeFactors n))
```

困难在于不能无限枚举 $x$。若 $n=\varphi(x)$，可由 $n$ 的质因数构造一个安全上界 $x\le \prod_{q\in\{1\}\cup\operatorname{factors}(n)}(q+1)$。因此 `tally n` 只需检查有限区间。

### 方法二：有限范围的有界版本

```haskell
highlyTotientNumbersUpTo :: Integer -> [Integer]
highlyTotientNumbersUpTo limit = go 1 0
  where
    go n best
      | n > limit = []
      | count > best = n : go (n + 1) count
      | otherwise    = go (n + 1) best
      where count = tally n
```

给定上界 `limit`，只生成不超过它的高欧拉函数数。对于只想看前几项的场景更可控。

### 方法三：基于 P34 版本（验证用）

