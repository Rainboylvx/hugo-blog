# P85 - 判断图同构

> Graph isomorphism
>
> 官方模块：`Problems.P85`　核心函数：`isomorphic`

## 题目描述

若存在顶点双射，使一张图中的边映射后恰好等于另一张图的边，两图同构。顶点编号本身没有意义。

## 实现

### 方法一：标准实现

先比较顶点数、边数和度数多重集，排除显然不可能的情况，再枚举顶点双射并检查边集。进一步优化时可以只枚举相同度数顶点之间的对应。

```haskell
import Data.List (permutations, sort)

isomorphic :: G -> G -> Bool
isomorphic left right
  | Set.size lvs /= Set.size rvs = False
  | Set.size les /= Set.size res = False
  | degrees left /= degrees right = False
  | otherwise = any preservesEdge candidateMappings
  where
    (lvs,les) = canonical left
    (rvs,res) = canonical right
    lvertices = Set.toList lvs
    rvertices = Set.toList rvs
    candidateMappings = map (Map.fromList . zip lvertices)
      (permutations rvertices)
    preservesEdge mapping = Set.map (rename mapping) les == res
    rename mapping (u,v) = normalize (mapping Map.! u, mapping Map.! v)

degrees graph = sort
  [Set.size (neighbors v graph) | v <- Set.toList (fst (canonical graph))]
```

度数过滤不会改变正确性，却能提前排除大量不可能情况。进一步优化可在构造映射时增量检查已映射边。

### 方法二：部分映射回溯

```haskell
import Data.List (sortOn)
import Data.Ord (Down(..))

isomorphicBacktracking :: G -> G -> Bool
isomorphicBacktracking left right
  | Set.size leftVertices /= Set.size rightVertices = False
  | Set.size leftEdges /= Set.size rightEdges = False
  | degrees left /= degrees right = False
  | otherwise = search orderedLeft Map.empty rightVertices
  where
    (leftVertices, leftEdges) = canonical left
    (rightVertices, rightEdges) = canonical right
    orderedLeft = sortOn
      (Down . Set.size . (`neighbors` left))
      (Set.toList leftVertices)

    search [] _ _ = True
    search (vertex:rest) mapping unused = any try candidates
      where
        wantedDegree = Set.size (neighbors vertex left)
        candidates =
          [candidate | candidate <- Set.toList unused
                     , Set.size (neighbors candidate right) == wantedDegree]

        try candidate =
          consistent candidate mapping
          && search rest (Map.insert vertex candidate mapping)
                         (Set.delete candidate unused)

        consistent candidate mapping = and
          [Set.member (normalize (other, vertex)) leftEdges
            == Set.member (normalize (mappedOther, candidate)) rightEdges
          | (other, mappedOther) <- Map.toList mapping]
```

先处理高度数顶点，只尝试映射到同度数的未用顶点。每增加一对映射，就比较它与所有已映射顶点之间的“有边/无边”关系；一旦不一致立即回溯，不再枚举剩余排列。

## 方法对比

| 方法 | 搜索单位 | 剪枝时机 |
|------|----------|----------|
| 全排列 | 完整双射 | 映射完成后比较全部边 |
| 部分映射回溯 | 一对顶点 | 每加入一对立即检查已映射关系 |

## 测试

```haskell
>>> isomorphic (toG (Paths [[1,2,3],[2,4]]))
...            (toG (Paths [[1,2,3],[1,4]]))
False
>>> isomorphic (toG (Paths [[1,2,3,1]]))
...            (toG (Paths [[4,5,6,4]]))
True
>>> isomorphicBacktracking (toG (Paths [[1,2,3,1]]))
...                        (toG (Paths [[4,5,6,4]]))
True
```

朴素枚举最坏为 $O(n!\cdot m)$，这正是本题困难所在。
