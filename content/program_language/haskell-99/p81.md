# P81 - 两点间所有无环路径

> Find all acyclic paths between two vertices
>
> 官方模块：`Problems.P81`　核心函数：`paths`

## 实现

DFS 的状态包含当前路径已访问顶点。每条递归分支都有自己的 `visited`，所以一条分支的选择不会污染其他候选路径。

### 方法一：DFS 回溯

```haskell
neighbors :: Vertex -> G -> Set.Set Vertex
neighbors v (G graph) = Map.findWithDefault Set.empty v graph

paths :: Vertex -> Vertex -> G -> [[Vertex]]
paths start target graph = go start Set.empty
  where
    go current visited
      | current == target = [[target]]
      | Set.member current visited = []
      | otherwise =
          [current : rest
          | next <- Set.toList (neighbors current graph)
          , rest <- go next (Set.insert current visited)]
```

### 方法二：按长度扩展路径（BFS）

```haskell
pathsBreadthFirst :: Vertex -> Vertex -> G -> [[Vertex]]
pathsBreadthFirst start target graph = expand [[start]]
  where
    expand [] = []
    expand frontier = completed ++ expand nextFrontier
      where
        completed = [path | path <- frontier, last path == target]
        unfinished = [path | path <- frontier, last path /= target]
        nextFrontier =
          [path ++ [next]
          | path <- unfinished
          , next <- Set.toAscList (neighbors (last path) graph)
          , next `notElem` path]
```

frontier 中的路径长度始终相同，因此结果按边数从少到多产生。路径自身就是访问集合，扩展时禁止加入已经出现的顶点。与 DFS 相比，它更早得到短路径，但会同时保存大量候选路径。

## 方法对比

| 方法 | 输出顺序 | 内存 |
|------|----------|------|
| DFS | 先完整探索一条分支 | 与递归深度和输出有关 |
| BFS | 按路径长度递增 | 需要保存整层候选路径 |

## 测试

```haskell
>>> let graph = toG (Paths [[1,2,3],[1,3,4,2],[5,6]])
>>> sort (paths 1 4 graph)
[[1,2,3,4],[1,2,4],[1,3,2,4],[1,3,4]]
>>> paths 2 6 graph
[]
>>> paths 1 1 graph
[[1]]
>>> sort (pathsBreadthFirst 1 4 graph) == sort (paths 1 4 graph)
True
```

简单路径数量在稠密图中可能是指数级，算法成本至少与输出总长度成正比。
