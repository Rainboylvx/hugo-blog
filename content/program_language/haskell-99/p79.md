# P79 - 用 Monad Transformer 求值后缀表达式

> Evaluate postfix notation with monad transformers
>
> 官方模块：`Problems.P79`　核心函数：`calculatePostfix`

## 状态、失败与日志

- `StateT [Integer]` 保存操作数栈；
- `MaybeT` 表示栈下溢、除零等失败；
- `Writer` 保存每一步的栈和操作符。

```haskell
import Control.Monad (mzero)
import Control.Monad.State
import Control.Monad.Trans.Maybe
import Control.Monad.Writer

data Operator = Negate | Add | Subtract | Multiply | Divide | Modulo
  deriving (Eq, Show)
data Element = Operator Operator | Operand Integer
  deriving (Eq, Show)

type History = [([Integer], Maybe Operator)]
type Calculation = MaybeT (StateT [Integer] (Writer History))
```

## 实现

### 方法一：`MaybeT + StateT + Writer`

```haskell
calculatePostfix :: [Element] -> (Maybe Integer, History)
calculatePostfix expression = (answer, history)
  where
    ((status, stack), history) =
      runWriter (runStateT (runMaybeT (mapM_ step expression)) [])
    answer = case (status, stack) of
      (Just (), [x]) -> Just x
      _              -> Nothing

step :: Element -> Calculation ()
step (Operand n) = modify (n:) >> record Nothing
step (Operator op) = apply op >> record (Just op)

record :: Maybe Operator -> Calculation ()
record op = get >>= \stack -> tell [(stack, op)]

pop :: Calculation Integer
pop = do
  stack <- get
  case stack of
    []     -> mzero
    x:rest -> put rest >> pure x

apply :: Operator -> Calculation ()
apply Negate = pop >>= \x -> modify ((-x):)
apply op = do
  b <- pop
  a <- pop
  value <- binary op a b
  modify (value:)

binary :: Operator -> Integer -> Integer -> Calculation Integer
binary Add      a b = pure (a + b)
binary Subtract a b = pure (a - b)
binary Multiply a b = pure (a * b)
binary Divide   _ 0 = mzero
binary Divide   a b = pure (a `div` b)
binary Modulo   _ 0 = mzero
binary Modulo   a b = pure (a `mod` b)
binary Negate   _ _ = mzero
```

### 方法二：用 `ExceptT` 保留失败原因

```haskell
import Control.Monad.Except

type CalculationE = ExceptT String (StateT [Integer] (Writer History))

calculatePostfixEither :: [Element] -> (Either String Integer, History)
calculatePostfixEither expression = (answer, history)
  where
    ((status, stack), history) =
      runWriter (runStateT (runExceptT (mapM_ stepE expression)) [])
    answer = case (status, stack) of
      (Left message, _)  -> Left message
      (Right (), [value]) -> Right value
      (Right (), _)       -> Left "expression did not leave one value"

stepE :: Element -> CalculationE ()
stepE (Operand n) = modify (n :) >> recordE Nothing
stepE (Operator operator) = applyE operator >> recordE (Just operator)

recordE :: Maybe Operator -> CalculationE ()
recordE operator = get >>= \stack -> tell [(stack, operator)]

popE :: CalculationE Integer
popE = do
  stack <- get
  case stack of
    []         -> throwError "stack underflow"
    value:rest -> put rest >> pure value

applyE :: Operator -> CalculationE ()
applyE Negate = popE >>= \value -> modify ((-value) :)
applyE operator = do
  right <- popE
  left <- popE
  value <- binaryE operator left right
  modify (value :)

binaryE :: Operator -> Integer -> Integer -> CalculationE Integer
binaryE Add      a b = pure (a + b)
binaryE Subtract a b = pure (a - b)
binaryE Multiply a b = pure (a * b)
binaryE Divide   _ 0 = throwError "division by zero"
binaryE Divide   a b = pure (a `div` b)
binaryE Modulo   _ 0 = throwError "modulo by zero"
binaryE Modulo   a b = pure (a `mod` b)
binaryE Negate   _ _ = throwError "unexpected binary negate"
```

状态栈和 Writer 历史保持不变，只把最外层失败效果从 `MaybeT` 换成 `ExceptT String`。这样仍然会在错误处短路，同时调用者能区分栈下溢、除零和最终栈元素数量错误。

## 方法对比

| 方法 | 失败信息 | 成功结果 |
|------|----------|----------|
| `MaybeT` | 只有 `Nothing` | `Maybe Integer` |
| `ExceptT String` | 具体错误文本 | `Either String Integer` |

## 测试

```haskell
>>> calculatePostfix [Operand 3, Operand 4, Operand 2,
...   Operator Subtract, Operator Multiply]
(Just 6,[([3],Nothing),([4,3],Nothing),([2,4,3],Nothing),
         ([2,3],Just Subtract),([6],Just Multiply)])
>>> fst (calculatePostfix [Operand 1, Operand 0, Operator Divide])
Nothing
>>> fst (calculatePostfixEither [Operand 1, Operand 0, Operator Divide])
Left "division by zero"
```

Transformer 的层次使三种效果保持独立：错误不会伪造结果，状态不需要手工穿线，历史也不会混入求值返回类型。
