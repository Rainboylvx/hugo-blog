# P47 - 通用逻辑门

> Universal logic gates
>
> 官方模块：`Problems.P47`　核心函数：`evaluateCircuit`, `buildCircuit`

## 题目描述

NAND 门是通用门——任意布尔函数都能用 NAND 门实现。给定电路描述（NAND 门列表），模拟电路输出；反之，给定一个布尔函数，构造出等价电路。

## 实现

### 方法一：递归计算电路

```haskell
evaluateCircuit :: [(Int, Int)] -> Bool -> Bool -> Bool
evaluateCircuit gates x y = last outputs
  where
    inputs = [(-1, x), (-2, y)]
    outputs = build 1 inputs gates
    build _ env [] = map snd (drop 2 (reverse env))
    build i env ((a,b):gs) =
      let value = not (lookupValue a env && lookupValue b env)
      in build (i + 1) ((i,value):env) gs

lookupValue :: Int -> [(Int, Bool)] -> Bool
lookupValue n ((k,v):rest)
  | n == k    = v
  | otherwise = lookupValue n rest
lookupValue _ [] = error "invalid gate reference"
```

`(-1)`、`(-2)` 分别代表两个输入。列表中第 $i$ 对 `(a,b)` 表示第 $i$ 个 NAND 门读取编号 $a$、$b$，最后一个门就是电路输出。

### 方法二：使用 fold 计算电路

```haskell
evaluateCircuit :: [(Int, Int)] -> Bool -> Bool -> Bool
evaluateCircuit gates x y = lookupResult
  where
    inputs = [(-1, x), (-2, y)]
    env = foldl step inputs (zip [1..] gates)
    step env (i, (a, b)) = (i, not (lookupValue a env && lookupValue b env)) : env
    lookupResult = case lookup (length gates) env of
                     Just v -> v
                     Nothing -> error "circuit evaluation failed"
```

用 `foldl` 替代显式递归，逐步更新环境变量。

两种求值写法采用相同的电路模型：方法一突出递归过程，方法二把每个门看成一次环境更新。

### 方法三：穷举构造 NAND 电路

```haskell
buildCircuit :: (Bool -> Bool -> Bool) -> [(Int, Int)]
buildCircuit f = head
  [ gates
  | gateCount <- [1..]
  , gates <- circuits gateCount
  , equivalent gates f
  ]
  where
    equivalent gates target =
      and [evaluateCircuit gates x y == target x y
          | x <- [False, True], y <- [False, True]]

circuits :: Int -> [[(Int, Int)]]
circuits gateCount = go 1
  where
    go gate
      | gate > gateCount = [[]]
      | otherwise =
          [ (left, right) : rest
          | left <- inputs gate
          , right <- inputs gate
          , rest <- go (gate + 1)
          ]

    inputs gate = [-2, -1] ++ [1 .. gate - 1]
```

第 $i$ 个门只能引用两个外部输入或编号小于 $i$ 的门，因此枚举出的电路一定无环。对四组输入比较真值表，就能找到与目标函数等价的第一个 NAND 电路。穷举速度不快，但对二元布尔函数是完整且可运行的构造方法。

## 测试

```haskell
>>> let orCircuit = [(-1,-1),(-2,-2),(1,2)]
>>> evaluateCircuit orCircuit True False
True
>>> and [evaluateCircuit (buildCircuit (&&)) x y == (x && y)
...     | x <- [False, True], y <- [False, True]]
True
```
