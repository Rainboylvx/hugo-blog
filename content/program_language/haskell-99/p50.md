# P50 - Huffman 编码

> Construct a Huffman encoding table
>
> 官方模块：`Problems.P50`　核心函数：`huffman`

## 算法

每个字符先是一棵带频率的叶子树。反复取出权重最小的两棵树合并，直到只剩根；向左边标 `0`、右边标 `1`，根到叶子的路径就是编码。

## 实现

```haskell
import Data.List (sortOn)

data HuffmanTree = Leaf Char | Branch HuffmanTree HuffmanTree

huffman :: [(Char, Int)] -> [(Char, String)]
huffman [] = []
huffman [(c,_)] = [(c,"0")]
huffman frequencies = codes (snd (head (build initial)))
  where
    initial = sortOn fst [(w, Leaf c) | (c,w) <- frequencies]

    build [tree] = [tree]
    build ((w,a):(v,b):rest) =
      build (sortOn fst ((w + v, Branch a b) : rest))
    build [] = []

codes :: HuffmanTree -> [(Char, String)]
codes = walk ""
  where
    walk prefix (Leaf c) = [(c, prefix)]
    walk prefix (Branch left right) =
      walk (prefix ++ "0") left ++ walk (prefix ++ "1") right
```

## 测试

```haskell
>>> let table = huffman [('a',45),('b',13),('c',12),('d',16),('e',9),('f',5)]
>>> all (\(_,code) -> not (null code)) table
True
```

同权重节点的合并顺序可能产生不同但同样最优的编码，因此实际验证应关注字符齐全、前缀无歧义和加权总长度。一个确定输入示例为：

```haskell
>>> huffman [('a',2),('b',1)]
[('b',"0"),('a',"1")]
```

## 复杂度

这里每次用列表重新排序，复杂度约 $O(n^2\log n)$；工程实现应使用优先队列，把合并过程降到 $O(n\log n)$。
