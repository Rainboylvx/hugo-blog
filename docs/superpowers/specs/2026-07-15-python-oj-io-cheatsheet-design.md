# Python OJ 输入输出速查文章设计

## 目标

在 `content/program_language/python/` 新增一篇可在比赛中快速查询、同时能帮助初学者理解代码的 Python OJ 输入输出文章。读者看到题目的输入格式后，应能直接定位到对应代码，并理解每一行代码为什么这样写。

文章不是单纯罗列模板，也不重复现有文章中的通用字符串操作。重点是 OJ 标准输入输出、常见题面格式和提交代码中的选择方法。

## 文件与栏目

- 新文章：`content/program_language/python/oj_input_output_cheatsheet.md`
- 标题：`Python OJ 输入输出速查：常见题面直接套用`
- front matter：沿用 Python 栏目习惯，使用 `date: 2026-07-15`、`draft: true`、`toc: true`，标签为 `Python`、`算法竞赛`、`输入输出`
- 栏目目录：在 `content/program_language/python/_index.md` 中新增链接
- 相关文章：与 `input_output_and_strings.md` 互相链接；新文章负责 OJ 场景速查，旧文章继续负责更完整的字符串处理和可测试解析函数

## 组织方式

采用按题面输入格式分类的结构。每种场景保持相同顺序：

1. 给出典型输入或输出格式；
2. 给出可直接提交或嵌入 `solve()` 的代码；
3. 逐句解释关键表达式；
4. 指出该场景最常见的错误。

这样既保留速查表的检索效率，又避免读者只会复制模板。

## 内容范围

文章覆盖以下场景：

1. 读取一个整数或字符串；
2. 一行读取固定数量的整数并解包；
3. 一行读取整数数组；
4. 读取多行数组和二维矩阵；
5. 第一行给出测试组数的多组数据；
6. 读取到 EOF；
7. 保留空格的整行字符串；
8. token 输入和整行文本混合时的处理边界；
9. 使用 `sys.stdin.buffer.readline` 处理大量行；
10. 使用 `sys.stdin.buffer.read` 一次读取所有整数；
11. 输出一个答案、输出数组和输出多行答案；
12. 用 `"\n".join(...)` 批量输出；
13. 根据输入规模和行结构选择读取方式；
14. 一份最小、可直接修改的 OJ 输入输出模板。

不展开排序、容器、生成器、算法实现或一般字符串 API；这些内容通过现有栏目文章承接。

## 代码与解释要求

- 示例使用真实的标准输入代码，不以 `StringIO` 或 `BytesIO` 代替主要示例。
- 需要验证的片段额外放入临时测试脚本运行，不把测试替身写成读者首先看到的用法。
- 解释 `split()`、`map(int, ...)`、解包、`*` 解包输出、`bytes` 可直接传给 `int` 等关键语义。
- 明确 `input()` 会去掉行末换行，而 `sys.stdin.buffer.readline()` 返回 `bytes`。
- 明确 `read()` 会丢失行结构，不适合空行或整行文本有意义的题目。
- 提醒不要混用已经读完整个 stdin 的 `read()` 与后续 `input()`。
- 二维列表使用列表推导式，避免把共享行的错误写法作为模板。
- 所有完整模板包含 `solve()` 和 `if __name__ == "__main__":`。

## 文章结尾

结尾提供一张选择表：

| 题面特征 | 推荐方式 |
|---|---|
| 输入较小、按行清晰 | `input()` |
| 行数很多、仍需保留行结构 | `sys.stdin.buffer.readline` |
| 全部是整数 token、换行无意义 | `sys.stdin.buffer.read` |
| 直到文件结束有多组数据 | 遍历 `sys.stdin.buffer` 或检查 `readline()` 返回值 |
| 需要读取包含空格的文本 | 按行读取并只去掉换行 |

随后给出最小模板，并链接到现有的输入输出与字符串文章、暴力代码大模板。

5. 确认没有修改任务范围之外的现有工作区内容。
