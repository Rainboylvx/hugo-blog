# Python 暴力验证文章重写设计

## 背景

`content/blog/python_stress_test.md` 当前以 `itertools` 和 Shell 对拍流程为主，标题同时承诺“暴力”和“对拍”，但没有系统回答一个更基础的问题：熟悉 C++ 算法竞赛、几乎没有 Python 经验的读者，需要掌握哪些 Python 知识，才能快速写出简单、可信的验证程序。

本次重写把重点从“如何搭建对拍流程”改为“如何用 Python 表达常见暴力模型”。对拍只作为验证函数的一种使用场景放在结尾，不展开脚本、进程管理或输出比较。

## 方案选择

### 方案一：Python 语法手册式

按列表、集合、推导式和标准库逐项介绍，便于查阅，但容易割裂 Python 知识与暴力验证任务之间的联系。

### 方案二：暴力模型驱动式（采用）

按点对、区间、子集、排列组合、回溯和状态搜索等暴力模型组织文章。每当一种模型第一次需要某个 Python 特性时，再用 C++ 选手熟悉的概念解释它。读者能够立即看到语法解决了什么问题。

### 方案三：完整案例驱动式

通过一道题从头写到尾，连贯性较好，但覆盖的枚举模型和 Python 工具有限，不适合作为日后复查的笔记。

## 目标读者

- 熟悉 C++ 基本语法、常用 STL 和算法竞赛中的暴力思想。
- 能理解循环、数组、集合、递归、DFS、BFS 和复杂度。
- 几乎没有 Python 使用经验，不默认理解切片、推导式、生成器、动态类型或可变对象语义。

文章不是完整的 Python 入门，也不负责讲解 DFS、BFS 等算法本身。重点是把读者已经理解的暴力思路快速翻译成 Python。

## 写作目标

文章读完后，读者应当能够：

1. 把“生成候选、判断合法、计算答案”拆成清晰的验证函数。
2. 使用 Python 枚举点对、区间、子集、排列、组合和多状态选择。
3. 在规则状态空间中使用 `itertools`，在依赖当前状态的搜索中使用 DFS、回溯或 BFS。
4. 使用列表推导式、生成器以及 `any`、`all`、`sum`、`min`、`max`、`next` 简洁表达判定和最优化。
5. 使用 `list`、`tuple`、`set`、`dict`、`deque` 和 `cache` 管理状态。
6. 避免浅拷贝、二维列表别名、状态未恢复、重复排列和空序列聚合等常见错误。

## 文章结构

### 1. 验证程序的基本结构

用“候选集合、合法性条件、目标函数”建立统一心智模型。说明验证代码以简单、直接、容易人工检查为第一目标，而不是追求能处理正式数据范围。

### 2. 枚举点对与区间

从 C++ 双重循环过渡到 Python 的 `range`、`enumerate` 和切片。示例覆盖无序点对、所有子数组和直接求区间答案。

### 3. 筛选候选与整体判定

通过列表推导式、生成器表达式、`zip`、`any` 和 `all` 表达筛选、相邻关系、存在性和全称条件。说明列表推导式会立即生成列表，而生成器按需产生元素。

### 4. 枚举子集

先给出 C++ 选手熟悉的位掩码写法，再给出 `itertools.product`。说明二者都枚举 $2^n$ 个状态，以及 `product` 如何扩展到每个位置有三种或更多选择的情况。

### 5. 枚举排列与组合

介绍 `permutations`、`combinations` 和必要时的 `combinations_with_replacement`。示例必须实际计算答案，不能只打印结果。说明返回值是元组、迭代器按需生成，以及重复输入会产生重复排列。

### 6. 选择依赖当前状态：DFS 与回溯

展示 `append -> dfs -> pop` 的基本模式，解释 Python 列表是可变对象，递归分支之间为什么必须恢复现场。给出需要根据当前状态动态决定下一步选择的完整例子。

### 7. 枚举状态图：BFS

使用 `collections.deque`、`set` 和 `tuple` 表示队列、已访问状态和可哈希状态。示例聚焦小状态空间的最短操作次数验证。

### 8. 消除重复搜索：记忆化

使用 `functools.cache` 把递归状态映射到答案。说明传给缓存函数的参数必须可哈希，因此经常需要把列表转换为元组。

### 9. 从候选得到答案

系统整理 `sum`、`min`、`max` 和 `next`：分别用于聚合、求最优值和找出第一个可行方案或反例。代码需要处理空候选集合，不能留下隐式异常。

### 10. Python 验证代码的常见陷阱

集中说明赋值不是复制、浅拷贝与深拷贝、`[[0] * m] * n` 的共享引用、可变状态恢复、列表不能直接放进集合、生成器只能消费一次、重复排列和空序列聚合。

### 11. 将验证函数用于检查正解

使用少量 `assert brute(data) == fast(data)` 和小规模随机数据展示验证函数的用途。只说明调用关系，不展开 Bash、编译、超时、进程退出码或自定义 checker。

## 单节写法

每个暴力模型尽量保持同一顺序：

1. 这一节要枚举什么。
2. C++ 中通常怎样表达。
3. Python 中需要认识的新语法或标准库。
4. 一段可独立运行的完整验证函数。
5. 状态数量、建议的小数据范围和最容易写错的地方。

代码以可读和可人工核对为第一目标。避免为了追求“一行代码”而隐藏状态变化；简洁表达确实更接近数学定义时才使用生成器和聚合函数。
