# Haskell 99 全部题解补全设计

## 背景

`content/program_language/haskell-99/` 已有 P01-P99 的文章文件和总目录。其中 P01-P20 已经写成完整题解，P21-P99 仍是只有 front matter、题目标题、空函数签名和空测试的模板。

本次工作保留 P01-P20 的现有正文，补全 P21-P99 共 79 篇文章，使整个系列可以连续阅读，并让每篇文章中的核心实现具有可验证性。

## 范围

### 包含

- 补全 `p21.md` 到 `p99.md`。
- 保持 `_index.md` 当前的十阶段分类和题目顺序。
- 修正补全过程中发现的题目名称、函数签名、导航链接或明显事实错误。
- 每篇文章提供题目描述、类型签名、思路、Haskell 实现、测试示例和参考链接。
- 对复杂题补充数据结构、算法步骤、复杂度或搜索剪枝说明。
- 完成后将对应文章的 `draft` 改为 `false`。

### 不包含

- 重写 P01-P20 的正文。
- 修改站点主题、菜单或全站样式。
- 为每道题建立独立 Cabal 包。
- 复制官方题解全文；文章使用自己的中文说明和独立整理的关键代码。

## 内容来源与准确性

每道题以文章中链接的 Ninety-Nine Haskell Problems 官方页面为首要依据，核对题意、函数名、输入输出和示例。总目录只负责学习路线，不作为函数签名的唯一来源。

对于存在多个流行版本的题目，文章明确采用当前官方页面的定义，不混用旧版题号。若仓库模板与官方页面冲突，修正文题、目录描述及相邻导航，使三者一致。

## 单篇文章结构

文章沿用 P11-P20 的格式：

1. YAML front matter：标题、日期、`draft: false`、`toc: true` 和现有标签风格。
2. H1 标题及英文题名、官方模块、核心函数。
3. 上一题和下一题导航，首尾题只显示存在的一侧。
4. `## 题目描述`：说明输入、输出、边界条件。
5. `## 函数签名`：给出实际使用的 Haskell 类型。
6. `## 思路`：解释递归关系、数据结构或算法选择。
7. `## 实现`：至少给出一种完整的核心实现；有学习价值时增加另一种实现。
8. `## 测试`：覆盖官方样例和至少一个关键边界条件。
9. 按题目需要增加复杂度、剪枝、表示转换或注意事项。
10. `## 要点总结` 和 `## 参考`。

代码块统一标记为 `haskell`。辅助类型、导入和核心辅助函数必须齐全，不能留下 `TODO`、省略号或无法理解的伪代码。大型题目允许省略与算法无关的命令行输入输出，但求解核心必须完整。

## 分阶段实施

按 `_index.md` 的现有阶段推进，每批完成后做一次局部检查：

1. P21-P28：组合与随机。
2. P29-P45：数论。
3. P46-P53：逻辑与编码。
4. P54-P69：二叉树。
5. P70-P79：多路树与 Monad。
6. P80-P89：图。
7. P90-P99：综合难题。

分批的目的不是降低完成范围，而是让题号、共享数据类型、导航和算法定义能在进入下一组前得到校验。

## 验证策略

### 结构检查

- P21-P99 不再含空的 ` ```haskell ` 代码块。
- 不含占位文本，例如 `TODO`、`-- >>>` 后无表达式或空的章节。
- 每篇仅保留一组页首导航和一组页尾导航。
- 相邻文章的上一题、下一题链接连续，P99 不出现下一题。
- 所有完成文章均为 `draft: false`。

### Haskell 检查

从每篇提取自包含的主实现和测试样例，按阶段组织临时验证文件，用本机可用的 GHC 工具链执行类型检查。依赖随机数等外部包的题目记录依赖；如果环境没有对应包，至少完成语法和类型层面的静态核对，并明确报告未执行项。

### Hugo 检查

运行 `hugo`，确保 front matter、Markdown、链接生成和代码块不会导致构建失败。

## 完成标准

- 每篇核心函数都有明确签名、完整实现和有效测试示例。

- 自动结构检查通过。
- Hugo 构建通过。
- Haskell 代码验证结果有清楚记录，任何受环境限制未执行的部分都会在交付说明中列出。

## 风险与处理

- **题目版本差异**：逐题以当前官方页面核对，发现冲突时同步修正目录和文章。
- **79 篇一次性修改难以审查**：按七批实施和验证，保持每批主题集中。
- **复杂搜索题代码过长**：保留完整求解核心，把解析和展示限制在理解算法所需的最小范围。
- **示例能读但不能编译**：优先提供自包含实现，并通过阶段验证文件检查导入、类型和边界条件。
